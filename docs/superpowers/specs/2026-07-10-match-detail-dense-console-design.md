# MATCH DETAIL 紧凑双栏驾驶舱设计

## 背景

WorldCup Oracle 的主要用户在桌面大屏上长时间盯盘，需要同时判断比赛基本面、模型预测、实时比赛数据和 Polymarket 盘口。当前 MATCH DETAIL 固定为三栏且高度接近全屏：赛前仍保留约 300px 的实时统计栏，中间预测卡垂直居中，造成明显的无效留白。

## 目标

- 让 1920×1080 桌面首屏同时呈现预测主体和盘口主体，不出现大片空洞。
- 保留现有比赛、模型、实时统计、天气和 Polymarket 数据能力。
- 赛前压缩无数据状态；开赛后自动提升实时统计的信息密度。
- 延续深色控制室风格，以 4/8/12/16px 为主要间距节奏。
- 1024px 以下保持自然阅读顺序和可用交互。

## 非目标

- 不修改 API、数据结构、盘口订阅或预测模型。
- 不增加新图表、投注操作或新的详情路由。
- 不重构与 MATCH DETAIL 无关的 Dashboard 区域。

## 布局

### 桌面（`lg` 及以上）

弹窗由固定三栏改为两栏：

- 左侧预测主区：约 60%，最小宽度为 0，允许内容安全收缩。
- 右侧盘口区：约 40%，保持不低于 420px，容纳概率曲线、深度梯和逐笔成交。
- 两栏均从顶部开始，不再对预测卡做垂直居中。
- 弹窗宽度上限维持 1440px；高度改为内容驱动，并限制在视口约 90% 内。内容超出时由对应栏内部滚动。

左侧顺序固定为：

1. 比赛预测 `FocusCard`
2. 实时统计 `LiveStats`

右侧保持 `LIVE 盘口 · ORDER BOOK` 标题和 `MatchDetail` 内容。

### 平板和移动端

低于 `lg` 时使用单列自然滚动：预测 → 实时统计 → 盘口。弹窗占满可用宽度，关闭按钮始终可见。关键内容不隐藏，只改变排列方式。

## 实时统计状态

`LiveStats` 增加仅供 MATCH DETAIL 使用的紧凑展示模式：

- 赛前：渲染单行状态条，包含模块名、状态和“开赛后显示控球/射门/角球/牌”等简短说明；不保留空数据容器高度。
- 加载中：保持同一行高，状态改为“加载中…”。
- 进行中或完场：展开为双列指标矩阵；每个指标仍显示主客数值和比例条，保留来源与比赛状态。
- 其他调用方继续使用现有默认样式，不受影响。

## 盘口密度

`MatchDetail` 增加控制台展示模式，仅在 MATCH DETAIL 弹窗中启用：

- 移除外层顶部空隙、重复背景、圆角和边框，避免卡片套卡片。
- 内部分组间距从 12px 收紧到 8px，保留状态、事件警报、概率曲线、深度梯、成交带和数据声明。
- 结果切换、深度梯和逐笔成交的交互与数据逻辑保持不变。

## 可访问性与交互

- `Escape`、遮罩点击和关闭按钮行为保持不变。
- 结果切换按钮继续支持键盘焦点。
- 语义颜色继续配合文字标签，不依赖颜色单独传达状态。
- 不增加持续动画；沿用现有实时点与减少动态效果策略。

## 验收

- 1920×1080：弹窗不再固定撑至 90vh；赛前实时统计为紧凑单行；预测主区顶部对齐；盘口主要内容首屏可见。
- 1366×768：无页面级横向溢出，各栏可独立滚动且关闭按钮可见。
- 1024×768、390×844：单列顺序正确，无横向溢出，盘口结果切换可操作。
- 赛前和完场比赛各验证一次：赛前统计折叠，完场统计可展开。
- 点击淘汰赛真实节点仍能打开 MATCH DETAIL；关闭和盘口 tab 交互正常。
- TypeScript 与生产构建通过；新增或修改文件不增加现有 lint 错误。
